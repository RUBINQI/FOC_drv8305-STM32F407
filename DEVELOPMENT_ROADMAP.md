# 开发路线（Development Roadmap）

本文件记录本项目从“能转”走向“工程可控、可量产化迭代”的完整开发路线。核心原则：FOC 的第一优先级不是公式，而是 **时序与同步**。

## 1. 总目标与约束

### 1.1 目标
- 面向“恒扭矩电钻”场景：以 Iq 控制扭矩，具备堵转保护、热保护、母线保护，运行稳定可复现

### 1.2 当前硬件与关键约束
- MCU：STM32F407（Cortex-M4F）
- 栅极驱动：DRV8305（SPI 配置 + 三相 PWM 输入）
- 位置传感器：AS5600（I2C）
- PWM：20kHz，中心对齐（周期 50µs）
- 现实约束：AS5600 的 I2C 角度读取不适合 20kHz 每周期同步读取，必须采用“两速率架构”

## 2. FOC 的实时控制时间轴（20kHz）

以中心对齐 PWM 为例，一个周期内理想的硬实时链路如下（以“触发点可调”为工程目标）：

- t = 0µs：PWM 边沿切换（MOSFET 开关噪声）
- t = 2~3µs：相电流趋于稳定
- t ≈ 3µs：TIM1 产生 ADC 触发（TRGO/TRGO2/OC 事件，延时可控）
- t ≈ 4µs：ADC 注入采样完成 → 进入 ADC ISR
- t = 4~12µs：ISR 内完成 FOC 流水线计算
- t ≈ 12µs：更新 TIM1 CCR（使用预装载，确保在安全时刻生效）
- t = 50µs：进入下一个周期

硬性要求：
- FOC 计算必须在本周期预算内完成（通常 <10µs）
- PWM → ADC → FOC → 更新 PWM 的同步关系必须可验证、可量化

## 3. 现状对照（按“功能”映射，不按函数名）

### 3.1 已具备
- TIM1：中心对齐 PWM、互补输出、死区与 MOE 使能链路已经跑通（见 [tim.c](file:///e:/STM32projects/FOC_stm32f4_as5600_drv8305/Core/Src/tim.c)、[pwm.c](file:///e:/STM32projects/FOC_stm32f4_as5600_drv8305/Core/Src/pwm.c)）
- DRV8305：SPI 初始化与寄存器配置已形成稳定上电序列（见 [drv8305.c](file:///e:/STM32projects/FOC_stm32f4_as5600_drv8305/Core/Src/drv8305.c)）
- AS5600：I2C 角度读取与基础封装已完成（见 [as5600.c](file:///e:/STM32projects/FOC_stm32f4_as5600_drv8305/Core/Src/as5600.c)）
- “角度 → 生成三相占空比 → 写 CCR”：已经能在中断里执行（见 [foc.c](file:///e:/STM32projects/FOC_stm32f4_as5600_drv8305/Core/Src/foc.c)）

### 3.2 与工程级 FOC 的核心差距
- 当前控制计算触发源是 TIM1 更新中断，而不是 “TIM1 触发 ADC 注入采样 → ADC ISR 内闭环计算”
- TIM1 的主触发输出目前为 RESET（尚未形成 PWM→ADC 的硬同步触发链路）
- ADC 目前未形成 Ia/Ib 的 20kHz 注入采样（且当前 ADC 仅见单通道配置）
- 角度读取在主循环轮询，无法与 20kHz 链路同步，存在采样错位与抖动风险

## 4. 开发路线（六阶段，工程可执行）

每一阶段都必须有明确“验收指标”，否则不能进入下一阶段。

### 阶段 A：功率级与 PWM 链路稳定
目标：稳定驱动三相 MOS，不因误配置导致过流/故障。
- TIM1：20kHz center-aligned，互补输出，死区，MOE，故障关断策略
- DRV8305：门极驱动参数、保护阈值、增益配置定型；维持“先 50% 占空比再 EN_GATE”的安全上电顺序

验收：
- 空载/假负载下波形正确，DRV8305 无异常 fault

### 阶段 B：建立硬实时“PWM→ADC→ISR→PWM”骨架（最关键分水岭）
目标：把控制从“定时中断”迁移到“ADC 注入完成中断”，并确保触发点位于噪声之后。
- TIM1：提供一个可控延时的 ADC 触发事件（建议用比较事件/触发输出，而不是更新瞬间）
- ADC：Injected 模式，双通道采样（Ia/Ib），触发源来自 TIM1，采样点固定在每周期相同位置
- 中断优先级：ADC ISR 高于 TIM1 更新 ISR；ISR 运行时间可测

验收：
- 示波器可验证：PWM 边沿后固定延迟触发 ADC；ADC ISR 周期稳定、无抖动尖峰

### 阶段 C：电流采样可用化（Offset/极性/重建）
目标：获得稳定的 Ia/Ib（先以 counts 表示也可以），确保符号、零偏与噪声可控。
- Offset 标定：PWM=50%（无净电压）下采样均值做零偏
- 极性/相序检查：用定向矢量或对齐动作验证 Ia/Ib 符号与电角度一致
- 故障策略：nFAULT/过流触发时，立刻关断 PWM/MOE，并冻结 PI 积分避免积分饱和

验收：
- 静止时电流接近 0 且不漂；动作时电流方向可预期

### 阶段 D：角度与 20kHz 控制的一致性（两速率架构）
目标：解决 AS5600 无法 20kHz 同步读取的现实约束，仍能给 20kHz ISR 提供连续可用的 θe。
- 低速任务（如 1kHz）：I2C 读取 θm（机械角）
- 高速任务（20kHz ISR）：使用 θm + 速度估计进行插值/外推，得到本周期的 θe（电角）
- 电角度：θe = θm × polepairs，并保持连续归一化

验收：
- 20kHz 内部 θe 连续、单调（匀速时），不会因 I2C 抖动出现阶跃

### 阶段 E：FOC 数学流水线落地到 ADC ISR
目标：把 FOC 变为固定、可度量、可优化的 ISR 流水线。

ADC ISR 内固定顺序（功能级）：
- 读取 Ia/Ib
- Clarke：Ia/Ib → Iα/Iβ
- Park：Iα/Iβ + sincos(θe) → Id/Iq
- PI：Id→0，Iq→Iq_ref（扭矩命令）
- Inverse Park：Vd/Vq → Vα/Vβ
- SVPWM：Vα/Vβ → duty_a/b/c
- 更新 CCR（预装载 + 安全生效）

验收：
- ISR 预算满足（例如 <10µs），电机可稳定转动且扰动下响应可预期

### 阶段 F：恒扭矩电钻控制（外环慢速，内环 20kHz）
目标：让“恒扭矩”成为系统级行为，而不是靠手调电压矢量。
- 20kHz：电流环（Id/Iq PI），Iq_ref 代表扭矩命令
- 低速（1kHz 或更低）：速度/扭矩策略、软启动、限流、温升模型、母线电压保护
- 抗堵转：Iq 限幅、时间积分保护、DRV fault 快速关断

验收：
- 扭矩可控；负载扰动下 Iq 可保持；堵转受控限流，不触发不可恢复故障

## 5. 重要设计原则（长期遵守）
- “时序第一”：先确定 PWM→ADC→ISR→PWM 的固定链路，再谈控制效果
- “两速率必需”：AS5600 的角度读取只能作为慢速观测/校正，20kHz 角度必须由估计器/插值器提供
- “可测量”：每个阶段都必须能用示波器/串口/统计指标证明“同步正确、预算满足”
